2. Question: How does PESTPP-OPT treat weights assigned to model outputs to which constraints are applied?
   Source: File: PEST Model-Independent Parameter Estimation: PEST++ a Software Suite for Parameter Estimation, Uncertainty Analysis, Management Optimization and Sensitivity Analysis, Section: 8.2 Using PESTPP-OPT
   Available Information: "It is important to note that weights assigned in the PEST control file to model outputs to which constraints are applied have no bearing on their operation, unless they are set to zero. If the weight associated with a constraint is set to zero, then the constraint is disabled."

3. Question: What requirement does PESTPP-OPT have regarding the application of constraints to model outputs?
   Source: File: PEST Model-Independent Parameter Estimation: PEST++ a Software Suite for Parameter Estimation, Uncertainty Analysis, Management Optimization and Sensitivity Analysis, Section: 8.2 Using PESTPP-OPT
   Available Information: "As presently programmed, PESTPP-OPT requires that at least one constraint be applied to a model output. Hence if the only constraints that are specified in a PEST control file that is read by PESTPP-OPT are those on individual decision variables through their bounds, and/or on prior information equations that feature those decision variables, PESTPP-OPT will cease execution with an appropriate message."
