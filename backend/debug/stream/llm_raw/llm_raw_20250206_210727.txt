Question: What is noptmax?
Raw Content:
**Direct Answer First:**
NOPTMAX is a control variable used in various PEST utility software, including PEST, PESTPP-GLM, and PESTPP-PSO, to determine the operation mode of a PEST run. It influences how PEST behaves in terms of model runs, objective function calculations, Jacobian matrix calculations, and parameter estimation processes. The value of NOPTMAX dictates whether PEST will perform a single model run to calculate the objective function, calculate the Jacobian matrix, or undertake parameter estimation. Specifically, setting NOPTMAX to 0 instructs PEST to perform a single model run and then stop, while setting it to -1 or -2 directs PEST to calculate the Jacobian matrix without proceeding to parameter estimation. A positive value of NOPTMAX signals PEST to engage in parameter estimation, utilizing initial parameter values that can be derived from various sources, such as Latin hypercube sampling. The PARREP utility allows for the optional setting of a new NOPTMAX value when creating a new PEST control file, indicating its flexibility in configuring subsequent PEST runs. Additionally, in the context of PESTPP-PSO's Pareto mode, NOPTMAX can be set to 0 to facilitate a large number of model runs based on an external initial-swarm file, showcasing its utility in diverse operational scenarios within PEST and its extensions.

   - **NOPTMAX=-1 or -2**: PEST calculates the Jacobian matrix, which comprises sensitivities of observations to parameters, but does not proceed to parameter estimation.
   - **NOPTMAX>0**: PEST undertakes parameter estimation, using initial parameter values that can be samples from a Latin hypercube or other sources.
   - **Optional new_noptmax in PARREP**: When using the PARREP utility, a new value for NOPTMAX can be optionally specified, allowing for the adjustment of PEST's operation mode in subsequent runs.
