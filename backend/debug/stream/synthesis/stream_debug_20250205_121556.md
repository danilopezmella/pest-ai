## What is lamforgive?

**Direct Answer:**
LAMFORGIVE is a control variable in PEST that dictates how model run failures during the lambda search process are handled. When set to "lamforgive", PEST ignores failed model runs and attempts further parameter upgrades using other values of the Marquardt lambda. If omitted or set to "nolamforgive", PEST ceases execution with an error message upon encountering a model run failure during this phase.

**Technical Format:**

- **Definition & Purpose:** LAMFORGIVE is designed to provide flexibility in how PEST responds to model run failures that occur while adjusting the Marquardt lambda during parameter estimation processes. Its primary function is to allow PEST to continue iterating through lambda values even if some model runs fail, rather than stopping the entire parameter estimation process.
  
- **Configuration Details:** LAMFORGIVE should be placed on the fifth line of the "control data" section of the PEST control file, following the JACUPDATE variable if present. It accepts two options: "lamforgive" to ignore failed model runs during lambda search, and "nolamforgive" to stop execution upon a failed model run.
  
- **Valid Options:** "lamforgive" (activates model run failure forgiveness), "nolamforgive" (deactivates model run failure forgiveness). The default setting is "nolamforgive" if LAMFORGIVE is omitted.
  
- **Impact & Best Practices:** Using "lamforgive" can be particularly useful in scenarios where model runs are likely to fail due to the experimental nature of the parameter values being tested. It allows PEST to explore a wider range of parameter values without being halted by intermittent model failures. However, it is important to monitor the process to ensure that the failures do not indicate a deeper issue with the model setup or the parameter space being explored.
  
- **Known Limitations:** While LAMFORGIVE can help maintain the continuity of the parameter estimation process, it may also lead to overlooking potential issues with the model or the parameterization that causes the failures. Careful consideration and analysis of the failed runs are advisable to ensure the integrity of the model calibration process.

**Documentation Reference:**
- **File:** PEST Model-Independent Parameter Estimation. User Manual Part I: PEST, SENSAN and Global Optimisers, **Section:** 4. The PEST Control File, **Subsection:** 4.2 Control Data Section
